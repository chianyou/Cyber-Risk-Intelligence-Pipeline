# Transformation Outputs

This document summarizes the transformation owner deliverables currently produced by the project.

## Core ETL Outputs

The project already produces these main transformed datasets:

- `data/curated/vulnerability_priority/vulnerability_priority_latest.csv`
- `data/curated/product_impact/dim_products_latest.csv`
- `data/curated/product_impact/bridge_cve_products_latest.csv`
- `data/curated/star_schema/fact_vulnerability_risk_latest.csv`

## Summary Datasets

The following summary datasets support the examples listed in the project role breakdown for the transformation owner.

### `fact_daily_new_cves_latest`

Path:

- `data/curated/transformation_summaries/fact_daily_new_cves_latest.csv`

Purpose:

- daily count of newly published CVEs

### `agg_cvss_severity_distribution_latest`

Path:

- `data/curated/transformation_summaries/agg_cvss_severity_distribution_latest.csv`

Purpose:

- CVSS severity distribution

### `agg_kev_hit_rate_by_cvss_severity_latest`

Path:

- `data/curated/transformation_summaries/agg_kev_hit_rate_by_cvss_severity_latest.csv`

Purpose:

- KEV hit rate grouped by CVSS severity

### `agg_epss_high_risk_by_vendor_product_latest`

Path:

- `data/curated/transformation_summaries/agg_epss_high_risk_by_vendor_product_latest.csv`

Purpose:

- distribution of high-EPSS vulnerabilities by vendor and product

Rule:

- `high EPSS` is currently defined as `epss_score >= 0.7`

### `agg_cwe_vendor_product_latest`

Path:

- `data/curated/transformation_summaries/agg_cwe_vendor_product_latest.csv`

Purpose:

- grouped view across CWE, vendor, and product
- includes CVE count, KEV hit rate, and average EPSS score

## Notes

- These outputs are generated by [build_transformation_summaries.py](src/transformation/build_transformation_summaries.py)
- All summary datasets are exported as both CSV and Parquet when `pyarrow` is available
